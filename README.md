# מחשבון עמלות בלוקצ'יין

</p>

מערכת לחישוב עמלות שליחה אשר מאתרת את עמלות הבלוקצ'יין ששולמו עבור כל עסקה, וממירה אותן לשקלים
---

## מה המערכת עושה?

לכל עסקת cashIn/cashOut שבקובץ, המערכת:

1. מאתרת את העסקה ברשת הבלוקצ'יין (BTC / ETH / USDT)
2. מחשבת את עמלת הרשת ששולמה (gas fee / mining fee)
3. ממירה לדולר לפי שער ביום העסקה
4. ממירה לשקלים לפי שער דולר/שקל באותו יום
5. מוסיפה גרסה עם מרווח של 6%
6. מייצאת הכל לקובץ פלט CSV

---

## דרישות מוקדמות (פעם אחת בלבד)

### 1. מפתח API של Etherscan

נדרש כדי לשלוף נתוני עסקאות ETH ו-USDT.

1. היכנסו לאתר [etherscan.io](https://etherscan.io) וצרו חשבון חינמי
2. לאחר ההרשמה, גשו ל: **My Account → API Keys → Add**
3. צרו מפתח חדש (שם לא חשוב)
4. העתיקו את המפתח שקיבלתם

### 2. הגדרת קובץ `.env`

בתיקיית הפרויקט, פתחו את הקובץ `.env` (או צרו אותו אם לא קיים) והוסיפו:

```
ETHERSCAN_API_KEY=הכניסו_כאן_את_המפתח_שלכם
```

## אופן השימוש

### שלב 1 — הכינו את קובץ הקלט

ייצאו קובץ CSV מהמערכת. הקובץ צריך לכלול את עמודות הסטנדרט של מערכת ה-ATM (כולל `txClass`, `txHash`, `toAddress`, `status` וכד').

שמרו את הקובץ, למשל: `2026-03-transactions.csv`

### שלב 2 — הריצו את הסקריפט

פתחו טרמינל בתיקיית הפרויקט והריצו:

```bash
python scripts/process_merchant_csv.py <קובץ_קלט> <קובץ_פלט>
```

**דוגמה:**

```bash
python scripts/process_merchant_csv.py 2026-03-transactions.csv output_march.csv
```

הסקריפט יתחיל לעבד ויציג התקדמות בזמן אמת:

```
[1/142] Processing: 9b20e191dd7895...
  [9b20e191dd7895] OK
[2/142] Processing: 0x011f5a949658e8...
  [0x011f5a949658e8] OK
...
```

### שלב 3 — קבלת התוצאות

בסיום, ייווצר קובץ `output_march.csv` עם העמודות הבאות:

| עמודה | תיאור |
|---|---|
| `source` | סוג עסקה: `cashin` או `cashout` |
| `hash` | מזהה עסקה בבלוקצ'יין |
| `blockchain` | רשת: BTC, ETH, USDT-ERC20 |
| `date` | תאריך העסקה |
| `wallet_address` | כתובת הארנק |
| `fee_crypto` | עמלה במטבע המקורי |
| `fee_usd` | עמלה בדולר |
| `usd_ils_rate` | שער דולר/שקל באותו יום |
| `fee_ils_standard` | עמלה בשקלים |
| `fee_ils_markup_6pct` | עמלה בשקלים + 6% מרווח |
| `error` | במקרה של שגיאה — פירוט הבעיה |

## טיפים ושאלות נפוצות

**השאלה:** הסקריפט נעצר באמצע — מה עושים?
**התשובה:** פשוט הריצו שוב עם אותה פקודה. הכלי מזהה אוטומטית אילו עסקאות כבר עובדו ומדלג עליהן (Resume mode).

---

**השאלה:** חלק מהעסקאות נכשלו (עמודת `error` לא ריקה)?
**התשובה:** הריצו שוב — עסקאות שנכשלו בגלל בעיית רשת ינסו שוב. אם הכשלון נמשך, ייתכן שהעסקה לא קיימת ברשת הבלוקצ'יין הצפויה.

---

**השאלה:** כמה זמן לוקח?
**התשובה:** כ-1–2 שניות לעסקה. לקובץ של 100 עסקאות — בערך 2–3 דקות.

---

**השאלה:** מאיפה מגיעים שערי ההמרה?
**התשובה:** שערי דולר/שקל מגיעים מ-Yahoo Finance. מחירי הקריפטו מגיעים מ-CoinGecko. שניהם נשלפים אוטומטית לפי תאריך העסקה — אין צורך להזין ידנית.
