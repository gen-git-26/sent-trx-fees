## מחשבון עמלות שליחה
---
 
## מה המערכת עושה?

לכל עסקת cashIn/cashOut שבקובץ, המערכת:

1. מאתרת את העסקה ברשת הבלוקצ'יין (BTC / ETH / USDT)
2. מחשבת את עמלת הרשת ששולמה (gas fee / mining fee)
3. ממירה לדולר לפי שער ביום העסקה
4. ממירה לשקלים לפי שער דולר/שקל באותו יום
5. מוסיפה גרסה עם מרווח של 6%
6. מייצאת הכל לקובץ פלט CSV

---

###  דו"ח סופי

בסיום, ייווצר קובץ `output_march.csv` עם העמודות הבאות:

| עמודה | תיאור |
|---|---|
| `source` | סוג עסקה: `cashin` או `cashout` |
| `hash` | מזהה עסקה בבלוקצ'יין |
| `blockchain` | רשת: BTC, ETH, USDT-ERC20 |
| `date` | תאריך העסקה |
| `wallet_address` | כתובת הארנק |
| `fee_crypto` | עמלה במטבע המקורי |
| `fee_usd` | עמלה בדולר |
| `usd_ils_rate` | שער דולר/שקל באותו יום |
| `fee_ils_standard` | עמלה בשקלים |
| `fee_ils_markup_6pct` | עמלה בשקלים + 6% מרווח |
| `error` | במקרה של שגיאה — פירוט הבעיה |

## שאלות נפוצות

**שאלה:** הסקריפט נעצר באמצע — מה עושים?
**תשובה:** פשוט הריצו שוב עם אותה פקודה. הכלי מזהה אוטומטית אילו עסקאות כבר עובדו ומדלג עליהן (Resume mode).

---

**שאלה:** חלק מהעסקאות נכשלו (עמודת `error` לא ריקה)?
**תשובה:** הריצו שוב — עסקאות שנכשלו בגלל בעיית רשת ינסו שוב. אם הכשלון נמשך, ייתכן שהעסקה לא קיימת ברשת הבלוקצ'יין הצפויה.

---

**שאלה:** כמה זמן לוקח?
**תשובה:** כ-1–2 שניות לעסקה. לקובץ של 100 עסקאות — בערך 2–3 דקות.

---

**שאלה:** מאיפה מגיעים שערי ההמרה?
**תשובה:** שערי דולר/שקל מגיעים מ-Yahoo Finance. מחירי הקריפטו מגיעים מ-CoinGecko. שניהם נשלפים אוטומטית לפי תאריך העסקה — אין צורך להזין ידנית.
